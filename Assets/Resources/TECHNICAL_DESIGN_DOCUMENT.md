## 1. 입력 시스템 구현


### 1.1 Touch 감지

**동작 방식**
```
1. 터치 시작 (touchstart)
   - 터치 위치 저장 (startX, startY)
   - 터치 시작 시간 기록 (startTime)
   - 터치 인덱스 기록(멀티 터치)
   - 터치한 객체 확인 (몬스터/캐릭터/빈 공간/UI)

2. 터치 종료 (touchend)
   - 터치 지속 시간 계산 (duration = endTime - startTime)
   - 터치 이동 거리 계산 (distance = √((endX-startX)² + (endY-startY)²))
   
   판단 로직:
   - duration < 300ms AND distance < 10px → Touch로 분류
   - duration ≥ 500ms → Hold로 분류
   - distance ≥ 30px → Slide로 분류
```

**구체적 구현 세부사항**

**Touch 판정**
- 터치 시간: 300ms 미만
- 이동 거리: 10px 미만
- 터치 지점에 몬스터가 있는가?
  - YES → HitType 0인 몬스터인가 확인 → 처치
  - NO → 캐릭터인가? → 특수 상호작용 (석화 해제 등)

**피드백**
- 터치 지점에 0.1초간 원형 이펙트 표시 (반지름 20px → 40px로 확대되며 페이드아웃)
- 색상: 흰색 반투명 (alpha: 0.8 → 0)
- 진동: 10ms 짧은 햅틱


### 1.2 Hold 감지

**동작 방식**
```
1. 터치 유지 중 (touchmove)
   - 현재 시간과 startTime 비교
   - 500ms 경과 시 Hold 상태 진입
   
2. Hold 상태 진입 시
    (몬스터 위에서 Hold 시)
   - 시각적 피드백 시작 (원형 게이지 애니메이션)
   - 0.5초 동안 원이 채워지는 애니메이션
   
3. Hold 완료 시
   - 몬스터 위에서 Hold → HitType 2 몬스터 속박
   - 캐릭터 위에서 Hold → 투사체 준비 모드
```

**구체적 구현 세부사항**

**Hold 게이지 애니메이션**
- 터치 지점 중심으로 원형 게이지 표시
- 내부 원: 투명 → 불투명 (0.5초 동안)
- 외부 테두리: 회전하는 점선 효과
- 색상: 
  - 몬스터 위: 하얀색
  - 캐릭터 위: 하늘색 + 하얀색

**Hold 성공 시**
- 몬스터 속박:
  - 몬스터 위로 커다란 손이 나타나 머리를 붙잡는 애니메이션 재생
  - 몬스터 이동 속도 0으로 설정
  - 2초 후 자동 해제 (손은 1.5초에 걸쳐 서서히 사라짐)
- 투사체 준비:
  - 캐릭터 기준 Hold 중인 위치의 반대편을 방향으로 하는 조준선(점선)을 표시하되 처음으로 닿는 몬스터까지만, Hold가 1초를 넘으면 적을 통과하는 조준선 생성  
  - 캐릭터 Touch 후 Slide로 새총을 당기는 애니메이션 실행, Hold로 고정하고 몸이 떨리는 애니메이션 실행(도중에 햅틱), Hold 해제 시 새총을 놓고 투사체를 발사하는 애니메이션 실행

### 1.3 Slide 감지

**동작 방식**
```
1. 터치 이동 중 거리 계산
   - 0.5초 안에 이동 거리 30px 이상 → Slide로 분류
   
2. Slide 경로 기록
   - 터치 지점들을 배열에 저장 (경로 추적)
   - 예: [{x: 100, y: 200}, {x: 105, y: 205}, ...]
   
3. Slide 종료 시
   - Slide 시간 동안 지난 지점들 사이의 존재하는 모든 몬스터 확인
   - Linecast 충돌 검사 수행
```

**구체적 구현 세부사항**

**일반 Slide (HitType 1)**

*경로 추적*
- 터치 이동 시 10px마다 위치 기록
- 기록된 점들을 연결하여 선분 생성
- 각 선분마다 충돌 검사 수행

*충돌 검사 (Linecast)*
```
의사코드:
for 각 선분 in Slide경로:
    for 각 몬스터 in 화면:
        if 선분이 몬스터의 충돌 영역과 교차:
            if 몬스터.hitType 포함 1:
                몬스터 처치()
```


**Slide Area Slide (HitType 4)**

*발동 조건*
- Slide 시작점이 Spawn Zone 1의 Outer line과 캐릭터 사이
- Slide 거리가 50px 이상

*공격 범위 계산*
```
1. Slide 방향 벡터 계산
   direction = (endX - startX, endY - startY)
   angle = atan2(direction.y, direction.x)

2. 부채꼴 범위 정의
   - 중심: 캐릭터 위치
   - 반지름: Spawn Zone 1의 Outer circle의 반지름과 동일
   - 각도 범위:
    1. 시계 방향으로 slide 시 : 첫 터치 지점 각도-80도 ~ 첫 터치 지점 각도+10도 (총 90도)
    2. 반시계 방향으로 slide 시 : 첫 터치 지점 각도-10도 ~ 첫 터치 지점 각도+80도 (총 90도)
   - 캐릭터 방향: 생성된 부채꼴의 중심

* 이미지 설명 참고 (C:\Users\PC\Slice_Shoot\Assets\Imagesbatswing.png)

3. 범위 내 몬스터 확인
   for 각 몬스터:
        몬스터_방향 = atan2(몬스터.y - 캐릭터.y, 몬스터.x - 캐릭터.x)
        if 시계 방향:
            if 첫 터치 지점 각도-80° ≤ 몬스터_방향 ≤ 첫 터치 지점 각도+10°:
                if 거리(캐릭터, 몬스터) ≤ 150px:
                    몬스터 처리()
        else:
            if 첫 터치 지점 각도-10° ≤ 몬스터_방향 ≤ 첫 터치 지점 각도+80°:
                if 거리(캐릭터, 몬스터) ≤ 150px:
                    몬스터 처리()
```

*밀쳐내기 효과*
- 맞은 몬스터에게 넉백 적용
- 넉백 방향: 캐릭터에서 몬스터 방향으로 일직선
- 넉백 거리: 30px
- 넉백 시간: 0.5초 (이징: easeOut)

*시각 효과*
- 야구방망이 스윙 애니메이션
  - 캐릭터 중심에서 Slide 방향으로 회전
  - 회전 각도: 90도 (0.2초에 걸쳐)
  - 방망이 궤적: 잔상 효과 (노란색, 반투명)

*추가 사항*
- 공격 시 1초간 공격 방향 응시 (도중에 회전 X)

### 1.4 Touch & Slide (투사체 발사)

**동작 방식**
```
1. 캐릭터를 Touch
   → 즉시 투사체 준비 모드 진입 (조준선 표시)
   → 캐릭터가 터치 반대 방향을 응시

2. Slide 후 바로 손가락 떼기 (터치 후 0.5초 미만)
   → 일반 투사체 발사 (터치 위치의 반대 방향)
   → 몬스터에 닿으면 사라짐

3. 터치 후 0.5초 이상 유지 후 떼기
   → 강화 투사체 발사 (관통 효과 + 사이즈 1.5배 Up)
   → 몬스터에 닿아도 관통, 3초 후 자동 소멸

* 강화 판정은 터치 시작부터의 총 시간 기준 (0.5초 이상 = 강화)
```

**구체적 구현 세부사항**

*조준 시스템*
- 캐릭터 Touch 즉시 터치 반대 방향으로 조준선 표시
- 캐릭터가 조준선 방향 (터치 반대 방향)을 응시
- 조준선 특성:
  - 캐릭터 중심 → 터치 위치의 반대로 연장선
  - 실선 (흰색 반투명, 폭 0.05)
  - 색상:
    - 일반: 흰색 반투명
    - 강화: 하늘색 반투명 (0.5초 경과 시 자동 전환)
  - 길이:
    - 일반: 몬스터에 닿았을 시 해당 몬스터까지만 표시
    - 강화: screen 가로 길이의 반 * 1.1

*발사 계산*
```
1. 발사 방향 계산
   direction = normalize(-(fingerEndWorldPos))
   // 손가락 최종 위치의 반대 방향

2. 투사체 생성
   projectile.position = 캐릭터.position (원점)
   projectile.velocity = direction * 속도(30)
   projectile.rotation = atan2(direction.y, direction.x)

3. 캐릭터 방향
   발사 후 0.3초간 발사 방향 응시 (LockFacing)
```

## 3. 몬스터 시스템 구현

몬스터 종류 : 일반, 보스
아케이드 단계 : 일반전 - 보스전
1. 일반전 : 일반 몬스터만 등장, 점수 획득 가능, 일정 점수 획득 시 보스전 진입
2. 보스전 : 보스 몬스터 추가, 점수 획득 불가, 보스 처치 시 스테이지 클리어




### 3.1 몬스터 생성 (Spawning)

**생성 위치 결정**
```
1. 생성 Zone 구획
* 이미지 참고 (Assets\Images\screen_sectioning.png)
    - Green Number: 각 Zone의 넘버
    - Red X: Screen의 세로 길이의 반
    - Zone 1 : 360도, Zone 4: 300도 ~ 60도, Zone 3: 60도 ~ 120도, Zone 2: 120도 ~ 240도, Zone 5: 240도 ~ 300도, Zone 6: 120도 ~ 240도, Zone 7: 300도 ~ 60도

2. 생성 위치
    - 일반: 몬스터 별 Data에 저장된 SpawnZone 중 랜덤으로 한 곳에서 생성 (ex. Zone 1,3,4 중 한 곳)
    - 보스: 몬스터 별 Data에 저장된 position에서 생성

3. 위치 검증
   - 일반: 생성 후 몬스터끼리 Collide되면 밀쳐내기 (RigidBody 활용)
   - 보스: 일반과 겹칠 수 있음. Touch, Slide, Hold의 우선순위는 일반 몬스터가 보스 몬스터보다 높음.

4. 몬스터 데이터 초기화
   (Scriptable Object 활용)
   monster = {
       byte speed: 타입별 속도,
       byte[] hitTypes: [0, 1, 2], // 예시
       byte monsterNum: 일반이면 1 이상(숫자 이하의 레벨에서 등장)의 자연수 / 보스면 0,
       byte bossNum: 보스 몬스터면 1 이상(몬스터 별로 겹치지 않게)의 자연수로 Level 별 출현 단계 표시 / 일반이면 0,
       byte currentHitIndex: 0,
       byte size: 40,
       string name: 이름
   }
```

**생성 타이밍**
```
일반전
- 일반 몬스터 사망 시 0.5초 후 일반 몬스터 소환
보스전
- 일반 몬스터 사망 시 1초 후 일반 몬스터 소환


화면 내 몬스터 수 제한:
- 레벨 1-2: 최대 4마리
- 레벨 3-4: 최대 5마리
- 레벨 5-6: 최대 6마리
- 레벨 7-8: 최대 7마리
- 레벨 9-10: 최대 8마리

레벨 별로 생성 몬스터의 구성이 달라짐 (ex. Lv1: 1,2,3 / Lv2: 1,2,4 / Lv3: 2,3,4,5...)

일반 몬스터 처치 시 바로 랜덤 소환
```

### 3.2 몬스터 이동

**직진 이동 (일반 몬스터)**
```
매 프레임마다:
1. 방향 벡터 계산
   direction = normalize(스크린 정중앙 - 몬스터 위치)

2. 위치 업데이트
   몬스터.x += direction.x * 몬스터.speed * deltaTime
   몬스터.y += direction.y * 몬스터.speed * deltaTime

3. 회전 (몬스터가 캐릭터를 바라봄)
   몬스터.rotation = atan2(direction.y, direction.x)

4. 주기적 재계산
- 도중에 위치가 변경될 시 다시 중앙을 향하게끔 조정 
```

### 3.3 몬스터 충돌 (서로 밀어내기)
```
일반 몬스터끼리는 RigidBody2D, Collider2D로 밀어내기
보스 몬스터는 충돌 없음

```

### 3.4 몬스터 레이어 시스템

**레이어 정의**

## 5. 충돌 감지 시스템

### 5.1 원형 충돌 (Circle Collision)

**기본 원리**
```
두 원의 충돌 검사:

function checkCircleCollision(A, B):
    거리 = √((A.x - B.x)² + (A.y - B.y)²)
    최소거리 = A.radius + B.radius
    
    return 거리 < 최소거리
```

**최적화**
```
제곱근 연산 피하기:

function checkCircleCollision(A, B):
    거리제곱 = (A.x - B.x)² + (A.y - B.y)²
    최소거리제곱 = (A.radius + B.radius)²
    
    return 거리제곱 < 최소거리제곱
```

### 5.2 선분 충돌 (Line-Circle Collision)

**Slide 경로 검사용**
```
function checkLineCircleCollision(lineStart, lineEnd, circle):
    1. 선분 벡터 계산
       line = lineEnd - lineStart
       length = magnitude(line)
    
    2. 가장 가까운 점 찾기
       t = dot(circle.position - lineStart, line) / (length²)
       t = clamp(t, 0, 1) // 0~1 범위로 제한
       closest = lineStart + t * line
    
    3. 거리 확인
       distance = magnitude(circle.position - closest)
       return distance < circle.radius
```

### 5.3 부채꼴 충돌 (Sector Collision)

**Slide Area Attack용**
```
function checkSectorCollision(center, angle, radius, arcAngle, point):
    1. 점까지 거리 확인
       distance = magnitude(point - center)
       if distance > radius:
           return false
    
    2. 각도 확인
       pointAngle = atan2(point.y - center.y, point.x - center.x)
       angleDiff = abs(pointAngle - angle)
       
       // 각도를 -π ~ π 범위로 정규화
       if angleDiff > π:
           angleDiff = 2π - angleDiff
       
       return angleDiff <= arcAngle / 2
```

### 5.4 충돌 처리 우선순위

**동시 충돌 시**
```
우선순위:
1. 투사체 - 몬스터
2. Slide - 몬스터
3. Touch - 몬스터
4. 몬스터 - 캐릭터 (게임 오버)

처리 순서:
매 프레임:
    1. 투사체 충돌 검사 및 처리
    2. Slide 충돌 검사 및 처리
    3. Touch 충돌 검사 및 처리
    4. 몬스터-캐릭터 충돌 검사
    5. 몬스터-몬스터 충돌 검사
```
